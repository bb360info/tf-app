# Быстрая тренировка без сезона (Standalone) — Проектирование Варианта C3

## 1. Отвязка Быстрой Тренировки от Сезона (Снятие Блокировки)

**Текущая проблема:** 
Компоненты (например, `ExerciseConstructor`, `AdHocWarmupStepBtn`, форма создания быстрой тренировки) запрашивают сезон по дате и блокируют UI (`"No season found for selected date"`), если сезона нет.

**Решение (Шаг 1):**
Изменить логику выборки сезона. Если сезона нет, UI больше не блокируется. 
При сабмите тренировки формируется payload для `training_plans`:
- `plan_type`: `"standalone"`
- `phase_id`: `null`
- `athlete_id`: `[ID атлета]`
- `start_date` / `end_date`: `[Выбранная дата]`
- `status`: `"published"` (чтобы атлет сразу ее видел)
- Возможна привязка группы (если тренер назначает на группу вне сезона).

---

## 2. Интеграция с UI Создания Фазы / Сезона (Консолидация)

**Проблема:** 
При создании новой фазы на даты графиков тренировок, система должна обнаруживать существующие `standalone` планы и предлагать их слить.

**Решение (Шаг 2):**
В форму создания/редактирования Фазы (`PhaseConfigurator` или аналогичный компонент) необходимо добавить новый этап — **"Конфликты расписания" (Schedule Conflicts)**.

1.  **Детектирование:** Когда тренер выбирает даты для новой Фазы (или двигает существующую), фронтенд делает запрос к `training_plans`, ища записи с `plan_type = 'standalone'` в этом диапазоне дат для атлета (или группы).
2.  **UI Конфликтов:** Если найдены такие планы, в форме появляется предупреждающий блок (Conflict Resolution Panel):
    *   *Найдено N тренировок вне сезона в эти даты.*
    *   Для каждой тренировки (или массово) предлагается выбор (Radio buttons / Select):
        *   **Merge & Override (Слить с заменой):** Тренировка станет `plan_type = 'override'`, получит `phase_id` новой фазы. Она заменит шаблонную тренировку фазы на этот день.
        *   **Keep Independent (Две сессии):** Тренировка остается `standalone` (`phase_id = null`). У атлета будет в этот день и системная тренировка по шаблону, и эта независимая.
        *   **Delete (Удалить):** Удалить старую тренировку.
3.  **Обработка на Backend:** Функция сохранения фазы (например, в `lib/pocketbase/services/plans.ts`) обрабатывает массив `conflictResolutions`.

## 3. Компоненты, требующие изменений

1.  **`src/components/dashboard/AthleteList.tsx` или где находится "Quick Workout" модалка:**
    *   Убрать жесткую проверку наличия сезона.
    *   Адаптировать функцию сохранения для поля `plan_type = 'standalone'`.

2.  **`src/components/season/PhaseCard.tsx` / компонент создания Фазы:**
    *   Добавить вызов API для поиска `standalone` планов.
    *   Добавить UI `ConflictResolutionPanel`.
    *   Добавить передачу резолюций в мутацию сохранения фазы.

3.  **`src/lib/pocketbase/services/plans.ts` (или аналогичный):**
    *   Обновить методы создания планов и фаз для поддержки логики трансформации `standalone` -> `override`.

## 4. Ограничения

*   Атлет будет видеть свои `standalone` тренировки в своем календаре так же, как и обычные сезонные (фронтенд календаря атлета просто рендерит все активные планы на дату).
