## QA Smoke Test Workflow

Используй этот воркфлоу перед деплоем или завершением трека для проверки работоспособности основных пользовательских путей.

1. Запусти `pnpm build` чтобы убедиться, что проект корректно собирается (Static Export).
```bash
pnpm build
```

2. Запусти локальный HTTP-сервер для проверки собранного продакшен-кода.
```bash
pnpx serve out -p 3000
```
> [!NOTE]
> Если сервер зависает на этом шаге, можешь пустить процесс в фон и продолжить со следующим шагом.

3. Используй `browser_subagent`, чтобы пройти базовые сценарии в браузере. Дай сабагенту следующую задачу:
```text
Открой http://localhost:3000 в браузере.
Проверь сценарий логина:
1. Выбери роль (Тренер/Атлет при необходимости)
2. Авторизуйся под тестовым аккаунтом (например, admin)
3. Убедись, что произошел редирект на Dashboard.
4. Проверь наличие основных карточек на Dashboard (тренировочные дни, чекин готовности).
5. Перейди на вкладку Training. Убедись, что отображается актуальный план.
Сделай скриншоты успешных состояний и верни полный текстовый отчет об отсутствии ошибок (или перечисли их, если найдешь).
```

4. Если сабагент нашёл ошибки — открой `CHANGELOG.md` и перечисли найденные баги в секции WIP, чтобы починить их до финального коммита.

5. После окончания QA-сессии останови запущенный сервер:
```bash
kill $(lsof -ti :3000)
```
