## QA Smoke Test Workflow

Используй этот воркфлоу перед деплоем или завершением трека для проверки работоспособности основных пользовательских путей.

1. Запусти preflight-проверки с ограниченным логом (token-safe output):

```bash
pnpm type-check 2>&1 | tail -40
pnpm lint 2>&1 | tail -40
pnpm build 2>&1 | tail -40
```

> [!NOTE]
> Показывай только последние строки и итоговый статус (exit code + ключевые ошибки), не вставляй полный лог.

1. Запусти локальный HTTP-сервер для проверки собранного продакшен-кода.

```bash
pnpx serve out -p 3000
```

> [!NOTE]
> Если сервер зависает на этом шаге, можешь пустить процесс в фон и продолжить со следующим шагом.

1. Запусти автоматизированные тесты против dev-сервера (быстрая проверка):

```bash
pnpm test:e2e
```

1. **ВАЖНО**: Перед финальным коммитом и деплоем проверь тесты против статической сборки (Production Parity):

```bash
pnpm test:e2e:preview
```

> [!NOTE]
> Это автоматически соберет проект (`pnpm build`) и запустит тесты на локальном сервере, имитирующем продакшен.

1. Если автоматизированных тестов недостаточно для новой фичи, используй `browser_subagent` для ручного исследования:

```text
Открой http://localhost:3000 в браузере.
[Опиши конкретные шаги для новой функциональности]
Сделай скриншоты и верни отчет.
```

1. Если найдены ошибки — зафиксируй их в секции WIP в `CHANGELOG.md` и исправь до коммита.

2. После окончания QA-сессии останови запущенный сервер:

```bash
kill $(lsof -ti :3000)
```
