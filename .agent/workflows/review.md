## Review Workflow

Запускай перед коммитом для финальной проверки изменённых файлов.

1. Узнай какие файлы были изменены:
```bash
git diff --name-only HEAD
```

2. Запусти проверку типов во всём проекте:
```bash
pnpm type-check
```

3. Запусти линтер (с автоматическим исправлением простых ошибок):
```bash
pnpm lint --fix
```

4. Проверь изменённые файлы на соблюдение дизайн-системы:
- Используются ли токены `var(--color-*)`, `var(--space-*)`? (Антипаттерн: хардкод HEX/RGB)
- Используются ли компоненты `lucide-react`? (Антипаттерн: эмодзи в UI)
- Нет ли хардкода строк в интерфейсе (должны использоваться i18n ключи `useTranslations`)?

5. Сформируй отчёт: **✅ OK** (всё чисто) или **⚠️ Issues** (что нужно поправить). Если есть проблемы, предложи их исправить ДО коммита.
