# UX Gap Report — Track 4.242

Дата: 2026-02-25  
Формат: только анализ (без внесения изменений в продуктовый код)

## Контекст

Проведён целевой аудит текущего состояния интерфейса после треков `4.20`–`4.241`, с фокусом на:

- несостыковки между UI и фактической функциональностью;
- отсутствующие кнопки/элементы для завершения пользовательских сценариев;
- UX-барьеры, увеличивающие когнитивную нагрузку или вероятность ошибок.

## Короткий вывод

Основная проблема не в визуале, а в «полурабочих» сценариях: пользователь видит активные элементы интерфейса, но часть действий либо ведёт в заглушки, либо не выполняет ожидаемую операцию. Это бьёт по доверию к системе и усложняет обучение продукту.

## Что усложняет UX (системно)

1. Ложные affordance: видимая кнопка не гарантирует действие.
2. Разрыв между навигацией и задачей: пользователь попадает в экран без завершающего шага.
3. Семантическая путаница метрик: «непрочитанные уведомления» маскируются под «pending review».
4. Неоднородная локализация: часть интерфейса следует i18n, часть остаётся hardcoded.
5. Mobile-first заявлен, но часть интерактивов не адаптирована под палец.

## Рекомендации для следующего агента

### P0 (в первую очередь)

1. Закрыть «битые» сценарии тренера:
   - реализовать содержимое `/training/review` (список + действия);
   - перевести бейдж `review` на отдельный источник данных `pending review`, а не `countUnread()`.
2. Устранить dead actions в шаблонах:
   - либо подключить реальные `onCopy/onEdit/onDelete`,
   - либо временно скрыть эти кнопки до реализации.
3. Активировать вход в уведомления из `More` (или убрать пункт полностью до готовности, без `(WIP)` в проде).

### P1

1. История плана: добавить минимум один завершающий action (`Restore version`) и подтверждение.
2. `Save PDF`: либо реализовать экспорт, либо убрать пункт из меню до готовности.
3. Уведомления: маппинг `message_key + params` на локализованные строки (fallback на `message` только при ошибке).
4. `Create first template`: привязать к реальному сценарию создания.

### P2

1. Привести touch targets к `>=44x44` в ключевых действиях карточек/логов/редакторов.
2. Для мобильного сценария сделать важные card actions видимыми по умолчанию (не только hover).
3. Убрать hardcoded EN строки из `ScoreCard` и аналогичных компонентов через `next-intl`.

## Критерии приёмки для UX-фазы

- Нет видимых кнопок, которые не выполняют бизнес-действие.
- Любой путь из нижней навигации и `More` приводит к завершённому сценарию.
- Бейджи отражают корректную сущность (review != unread notifications).
- Ключевые интерактивы проходят touch-check (`44x44`) на мобильном.
- RU/EN/CN не содержат смешения локализованных и хардкод-строк в целевых экранах.
