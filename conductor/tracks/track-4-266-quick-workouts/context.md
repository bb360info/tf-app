# Track 4.266: Context-Aware Quick Workouts (Option C3)

## Контекст (Как было до)

Ранее форма **Quick Workout** (`QuickWorkout.tsx`) требовала наличия активного сезона у атлета на выбранную дату. Если сезона не было, метод сохранения `ensurePlanAndAssign` выбрасывал ошибку: `"noSeasonForDate"`. Это мешало тренеру создавать разовые тренировки или ознакомительные сессии до формального старта сезона.

В БД таблица `training_plans` уже поддерживает планы без привязки к фазе:

- `plan_type` ('phase_based', 'standalone', 'override')
- `phase_id` (опционален)

## Задача

Позволить создание `standalone` тренировок, если сезон отсутствует, и реализовать правильную обработку "конфликтов" (перекрытий) при последующем создании сезона (Вариант C3).

## Архитектурное решение (Вариант С3: "Context-Aware Hybrid")

1. **Быстрая тренировка без сезона:** Сохраняется как `plan_type = 'standalone'`, `phase_id = null`.
2. **Создание сезона (Конфликт):** Когда тренер создает новый сезон (через `SeasonWizard.tsx`), система проверяет, пересекаются ли даты нового сезона с существующими `standalone` планами выбранного атлета (с помощью новой функции `getStandalonePlansForPeriod`).
3. **Разрешение конфликта:** Тренеру в `SeasonWizard` предлагаются 3 варианта для каждого конфликта (или оптом):
   - **Merge & Override (Включить в сезон):** Тренировка становится перекрытием. Меняем `plan_type` на `override`, присваиваем `phase_id` = ID новой фазы.
   - **Keep Independent (Оставить независимо):** Тренировка остается `standalone` и существует параллельно с планом фазы (например, независимая утренняя зарядка).
   - **Delete (Удалить):** Удаление (`deleted_at`).

## Затрагиваемые файлы

- `src/lib/pocketbase/services/plans.ts` (Новые функции для `standalone`)
- `src/components/training/QuickWorkout.tsx` (Удаление блокировки "нет сезона")
- `src/components/training/SeasonWizard.tsx` (Интерфейс разрешения конфликтов)
- Файлы переводов `messages/*/common.json` (новые ключи для UI конфликтов)
