# Context: Track 4.244 - Historical PR Onboarding

**Проблема:**
В текущем релизе (Gate 4.243) мы убрали ручной ввод Personal Records. Вместо этого они вычисляются из официальных результатов соревнований. Однако новым пользователям (атлетам) больше негде задать свой базовый результат в процессе онбординга. Это приводит к пустым графикам прогресса и "My PR: —".

**Решение:**
Не создавать изолированную "легкую" форму для PR, а использовать полноценный функционал создания прошлых соревнований (`pastForm`) из `CompetitionsHub`.
Атлет с дашборда нажимает специальный CTA (Call to Action) "Установи точку отсчета", переходит на страницу соревнований с автоматически открытой панелью регистрации прошлого старта, вносит результат, город, ссылку, и дату. Как только он нажимает "Создать", система генерирует соревнование, привязывает атлета, выставляет официальный результат, и на экране сразу появляется готовая `CompetitionCard`, куда атлет может загрузить видео своего рекордного прыжка.

**Затронутые файлы:**

- `pb_schema` (API Rules для разрешений)
- `src/components/dashboard/AthleteDashboard.tsx` (CTA Виджет)
- `src/components/competitions/CompetitionsHub.tsx` (Доступ к pastForm для атлетов и URL parameters)
- `src/lib/pocketbase/services/competitions.ts` (Возможно, вспомогательная функция)
